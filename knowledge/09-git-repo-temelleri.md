# Git Repo Temelleri: README, Badge, Lisans

## README.md Kullanimi

- Her repo'nun ana klasorunde bulunan `README.md`, projeye giren ilk kisinin okudugu dosyadir.
- Iyi bir README genelde sunlari icerir: projenin ne oldugu, ozellikleri, kurulum adimlari, kullanilan teknolojiler, katkida bulunma yontemi ve lisans bilgisi.
- Bu repodaki [README.md](/Users/Q/Documents/HelloAI-Universite/README.md) dosyasi buna bir ornektir.

## Badge (Rozet) Kullanimi

- Badge'ler, README'nin ustunde yer alan kucuk gorsel etiketlerdir (ornek: build durumu, versiyon, lisans, kullanilan teknoloji).
- Genelde [shields.io](https://shields.io) gibi servislerle otomatik olusturulur ve Markdown'a resim linki olarak eklenir.
- Amaci: Proje hakkinda hizli, gorsel bir ozet sunmak (ornek: "bu proje aktif mi?", "hangi lisansla dagitiliyor?", "testler geciyor mu?").

## Lisanslar

- Lisans, projenin baskalari tarafindan nasil kullanilabilecegini, degistirilip degistirilemeyecegini ve ticari kullanim haklarini belirler.
- Yaygin acik kaynak lisanslari: **MIT** (cok esnek, neredeyse serbest kullanim), **Apache 2.0** (patent koruma maddeleri ile), **GPL** (turev projelerin de acik kaynak kalmasini zorunlu kilar).
- Bir repo'ya lisans eklemek icin ana klasore `LICENSE` dosyasi eklenir; Github repo olustururken de hazir sablonlardan secilebilir.
- Lisans belirtilmemis bir proje varsayilan olarak "tum haklari sakli" kabul edilir; baskalari izinsiz kullanmamalidir.
