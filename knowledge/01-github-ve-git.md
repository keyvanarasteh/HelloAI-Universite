# Github ve Git

## Github Hesabi

- Her ogrenci kendi Github hesabini acmali.
- Profil, gelecekte is basvurularinda bir "yazilim CV'si" gibi calisir; duzenli ve gercek katkilarla dolu bir profil onemlidir.

## Git Kullanimi

- **Teamwork:** Birden fazla kisinin ayni proje uzerinde calisirken git ile nasil coakisma yasamadan ilerleyecegi.
- **Changelog:** Commit mesajlarinin degisiklikleri anlasilir sekilde ozetlemesi, projenin gecmisinin okunabilir olmasi.
- **Branch kullanimi:** Yeni bir ozellik veya duzeltme icin ana koddan (`main`) ayri bir dal (`branch`) acip orada calismak, isi bitince `main`'e birlestirmek (`merge`).
- **Git CV onemi:** Github'daki commit gecmisi, projeler ve katkilar bir gelistiricinin tecrubesini gosteren onemli bir gostergedir. Issveren ve is ortaklari bu gecmise bakar.

## Fake Commit Konusu

- Github'da commit tarihini/sayisini yapay olarak sisirmek (`fake commit atmak`) teknik olarak mumkundur (ornegin commit tarihini degistirerek).
- Bu davranis algilanabilir: commit gecmisindeki tutarsizliklar, gercek kod degisikligi olmayan bos commitler, anormal commit yogunlugu gibi izler fark edilebilir.
- **Bu kesinlikle yapilmamalidir.** Sahte katki gecmisi hem etik degildir hem de fark edildiginde guven kaybina yol acar.
